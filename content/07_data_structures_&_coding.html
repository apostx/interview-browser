Big-O & Complexity AnalysisBig-O & komplexitás-elemzés

How runtime and memory grow as input grows — the language interviewers use to compare solutions. You state it for every solution you propose, before and after optimizing.Hogyan nő a futásidő és a memória, ahogy az input nő — ez a nyelv, amin az interjúztatók összehasonlítják a megoldásokat. How do you analyze a loop quickly?Hogyan elemzel gyorsan egy ciklust?

  • Count how many times the work runs relative to n. Sequential loops add (O(n) + O(n) = O(n)); nested loops multiply (O(n) × O(n) = O(n²)). Drop constants and lower-order terms — O(2n + 5) is O(n).Számold meg, hányszor fut a munka n-hez képest. Az egymás utáni ciklusok összeadódnak (O(n) + O(n) = O(n)); az egymásba ágyazottak szorzódnak (O(n) × O(n) = O(n²)). Dobd el a konstansokat és alacsonyabb rendű tagokat — az O(2n + 5) az O(n).

What is amortized complexity?Mi az amortizált komplexitás?

  • The average cost per operation across a sequence, when occasional operations are expensive. Pushing to a dynamic array is amortized O(1): most pushes are constant, and the occasional resize that copies everything averages out.A műveletenkénti átlagos költség egy sorozaton át, amikor egyes műveletek drágák. Egy dinamikus tömbbe push-olás amortizált O(1): a legtöbb push konstans, és az alkalmankénti átméretezés, amely mindent átmásol, kiátlagolódik.

Why does space complexity matter separately?Miért számít külön a tárkomplexitás?

  • Interviewers often ask for an O(1)-space version after you give an O(n)-space one. Remember the recursion call stack counts as space — a recursive DFS on a tree is O(h) space, not O(1).Az interjúztatók gyakran kérnek O(1)-tárú verziót, miután adtál egy O(n)-tárút. Ne feledd, a rekurziós hívási verem is tár — egy rekurzív DFS egy fán O(h) tár, nem O(1).

Watch outBuktatóDon't say “O(n) because there's one loop” without checking what's inside it. A .includes() or .indexOf() inside a loop is a hidden O(n), making the whole thing O(n²) — this is the single most common accidental quadratic.Ne mondd, hogy „O(n), mert egy ciklus van”, anélkül hogy megnéznéd, mi van benne. Egy .includes() vagy .indexOf() a cikluson belül rejtett O(n), amitől az egész O(n²) lesz — ez a leggyakoribb véletlen négyzetes eset.

Arrays & StringsTömbök & stringek

Contiguous, index-addressable sequences — the default structure, and the one most interview problems are phrased over. Strings behave like immutable arrays in JavaScript.Folytonos, indexelhető sorozatok — az alapértelmezett szerkezet, és az, amire a legtöbb interjúfeladat épül. A stringek JavaScriptben immutábilis tömbként viselkednek.

access O(1)elérés O(1)search O(n)keresés O(n)insert/delete at front O(n)beszúrás/törlés elöl O(n)push/pop O(1)*push/pop O(1)*

Why is shift/unshift O(n)?Miért O(n) a shift/unshift?

  • Removing or inserting at index 0 requires shifting every remaining element by one position. Use push/pop (the end) when order permits, or a deque-like approach with an index pointer instead of mutating the front.A 0. indexen való törlés vagy beszúrás minden maradék elem eggyel való eltolását igényli. Használj push/pop-ot (a végén), ha a sorrend engedi, vagy deque-szerű megközelítést index-mutatóval a front mutálása helyett.

What string operations hide a cost in JS?Mely string-műveletek rejtenek költséget JS-ben?

  • Strings are immutable, so s += c in a loop creates a new string each time — build an array and join('') at the end. slice/substring copy, so repeated slicing inside a loop is quietly O(n²).A stringek immutábilisak, szóval az s += c egy ciklusban minden alkalommal új stringet hoz létre — építs tömböt és join('') a végén. A slice/substring másol, szóval az ismételt szeletelés cikluson belül csendben O(n²).
Building a string — the two versionsString építése — a két verzióO(n²) → O(n)
// ✗ O(n²) — new string every iteration
let out = '';
for (const c of chars) out += c;

// ✓ O(n) — one join at the end
const parts = [];
for (const c of chars) parts.push(c);
const out2 = parts.join('');

Watch outBuktatóIn JS, array.splice, shift and unshift are O(n) but look as cheap as push. Using shift() to dequeue in a BFS loop turns an O(V+E) traversal into O(V²) — use an index pointer instead.JS-ben az array.splice, shift és unshift O(n), de olyan olcsónak tűnnek, mint a push. A shift() használata dequeue-ra egy BFS ciklusban O(V+E) bejárást O(V²)-re változtat — használj helyette index-mutatót.

Hash Maps & SetsHash map-ek & set-ek

Key → value lookup in expected O(1) via hashing. The single highest-leverage structure in interviews: it converts a nested-loop search into a single pass.Kulcs → érték lookup várhatóan O(1)-ben hashelés útján. A legnagyobb hatású szerkezet interjúkon: egy egymásba ágyazott ciklusú keresést egyetlen végigmenéssé alakít.

get/set/has O(1) avgget/set/has O(1) átl.O(n) worst caseO(n) legrosszabbspace O(n)tár O(n)

Map vs plain object in JavaScript?Map vs sima objektum JavaScriptben?

  • Map accepts any key type (including objects), preserves insertion order, has a real .size, and is safe from prototype-key collisions. Prefer Map for algorithm work; objects are fine for fixed string keys.A Map bármilyen kulcstípust elfogad (objektumot is), megőrzi a beszúrási sorrendet, van valódi .size-a, és védett a prototípus-kulcs ütközésektől. Algoritmusokhoz a Map-et részesítsd előnyben; objektumok rendben vannak fix string kulcsokhoz.

When does O(1) degrade?Mikor romlik el az O(1)?

  • On heavy hash collisions all keys land in one bucket, degrading to O(n). In practice with built-in hashing this is a theoretical concern — but it's the right answer when asked for the worst case.Erős hash-ütközéseknél minden kulcs egy bucketbe kerül, O(n)-re romolva. A gyakorlatban a beépített hasheléssel ez elméleti aggály — de ez a helyes válasz, ha a legrosszabb esetet kérdezik.
Two Sum — the canonical hash-map winTwo Sum — a kanonikus hash-map nyereségO(n) time / O(n) space
function twoSum(nums, target) {
  const seen = new Map();          // value → index
  for (let i = 0; i < nums.length; i++) {
    const need = target - nums[i];
    if (seen.has(need)) return [seen.get(need), i];
    seen.set(nums[i], i);
  }
  return [];
}

Watch outBuktatóThe trade is always space for time — say so out loud. Also, objects/arrays as Map keys compare by reference, not value: two identical arrays are different keys. Serialize (e.g. JSON.stringify or a delimiter-joined string) when you need value equality.A csere mindig tár időért — mondd ki hangosan. És az objektumok/tömbök mint Map-kulcsok referencia szerint hasonlítódnak, nem érték szerint: két azonos tömb két különböző kulcs. Szerializálj (pl. JSON.stringify vagy elválasztóval összefűzött string), ha érték-egyenlőség kell.

Stacks & QueuesVeremek & sorok

Order-restricted collections: a stack is last-in-first-out (LIFO), a queue is first-in-first-out (FIFO). Both give O(1) insert and remove at their designated end.Sorrend-korlátozott gyűjtemények: a verem last-in-first-out (LIFO), a sor first-in-first-out (FIFO). Mindkettő O(1) beszúrást és eltávolítást ad a kijelölt végén.

push/pop O(1)push/pop O(1)peek O(1)peek O(1)space O(n)tár O(n)

What signals a stack problem?Mi jelez verem-feladatot?

  • Matching or nesting (brackets, tags), undo/backtracking, evaluating expressions, and “most recent unresolved thing” — including the monotonic-stack family (next greater element, largest rectangle).Párosítás vagy egymásba ágyazás (zárójelek, tagek), undo/backtracking, kifejezés-kiértékelés, és a „legutóbbi megoldatlan dolog” — beleértve a monoton verem családot (next greater element, legnagyobb téglalap).

How do you implement an efficient queue in JS?Hogyan implementálsz hatékony sort JS-ben?

  • Not with shift() (O(n)). Keep an array plus a head index and advance the index on dequeue — O(1) per operation, at the cost of not reclaiming memory until you compact.Nem shift()-tel (O(n)). Tarts egy tömböt plusz egy head indexet, és léptesd az indexet dequeue-nál — O(1) műveletenként, azon az áron, hogy nem szabadítasz fel memóriát, amíg nem tömörítesz.
O(1) queue via head pointerO(1) sor head-mutatóvalenqueue/dequeue O(1) amortized
class Queue {
  #items = []; #head = 0;
  enqueue(x) { this.#items.push(x); }
  dequeue() {
    if (this.#head >= this.#items.length) return undefined;
    const x = this.#items[this.#head++];
    // compact occasionally to free memory
    if (this.#head > 50 && this.#head * 2 > this.#items.length) {
      this.#items = this.#items.slice(this.#head);
      this.#head = 0;
    }
    return x;
  }
  get size() { return this.#items.length - this.#head; }
}

Watch outBuktatóUsing array.shift() as a queue is the most common hidden-quadratic bug in interview code — it looks like the obvious dequeue but copies the whole array each call.Az array.shift() sorként használata a leggyakoribb rejtett-négyzetes bug az interjúkódban — a kézenfekvő dequeue-nak tűnik, de minden hívásnál lemásolja az egész tömböt.

Linked ListsLáncolt listák

Nodes connected by pointers — O(1) insert/delete given a node reference, but no random access. Interview-relevant mainly for pointer manipulation problems.Mutatókkal összekötött node-ok — O(1) beszúrás/törlés egy node-referencia birtokában, de nincs véletlen elérés. Interjú-szempontból főleg mutató-manipulációs feladatokhoz releváns.

insert/delete at node O(1)beszúrás/törlés node-nál O(1)access/search O(n)elérés/keresés O(n)space O(n)tár O(n)

What are the two essential techniques?Mi a két alapvető technika?

  • Fast/slow pointers (Floyd): detect a cycle, find the middle, find the k-th from the end. Dummy head node: removes the special case of modifying the first element, simplifying insert/delete code substantially.Gyors/lassú mutatók (Floyd): ciklus észlelése, a közép megtalálása, a k-adik a végétől. Dummy head node: megszünteti az első elem módosításának speciális esetét, jelentősen egyszerűsítve a beszúró/törlő kódot.
Reverse a linked list — iterativeLáncolt lista megfordítása — iteratívO(n) time / O(1) space
function reverse(head) {
  let prev = null, cur = head;
  while (cur) {
    const next = cur.next;   // save before overwriting
    cur.next = prev;         // flip the pointer
    prev = cur;              // advance
    cur = next;
  }
  return prev;               // new head
}

Watch outBuktatóAlways save cur.next before you overwrite it, or you lose the rest of the list. And check for null at every dereference — most linked-list bugs are missing null guards at the head or tail.Mindig mentsd el a cur.next-et, mielőtt felülírod, különben elveszted a lista többi részét. És ellenőrizd a null-t minden dereferálásnál — a legtöbb láncolt lista bug hiányzó null-őr a fejnél vagy a végénél.

Trees & Binary Search TreesFák & bináris keresőfák

Hierarchical nodes with a single root and no cycles. A binary search tree keeps left < node < right, giving O(log n) search when balanced.Hierarchikus node-ok egyetlen gyökérrel, ciklusok nélkül. Egy bináris keresőfa fenntartja a bal < node < jobb rendet, ami kiegyensúlyozott állapotban O(log n) keresést ad.

BST search O(log n) balancedBST keresés O(log n) kiegyensúlyozvaO(n) if degenerateO(n), ha elfajultspace O(h) recursiontár O(h) rekurzió

The traversal orders and when to use each?A bejárási sorrendek és mikor melyik?

  • In-order (left, node, right) — yields a BST's values in sorted order. Pre-order (node, left, right) — copying/serializing a tree. Post-order (left, right, node) — deleting, or computing a value from children first. Level-order (BFS) — anything depth- or layer-related.In-order (bal, node, jobb) — egy BST értékeit rendezett sorrendben adja. Pre-order (node, bal, jobb) — fa másolása/szerializálása. Post-order (bal, jobb, node) — törlés, vagy érték számítása a gyerekekből előbb. Level-order (BFS) — bármi mélység- vagy szint-vonatkozású.

What makes a tree balanced, and why care?Mitől kiegyensúlyozott egy fa, és miért érdekes?

  • Subtree heights differ by at most a constant, keeping height O(log n). An unbalanced BST built from sorted input degenerates into a linked list — search becomes O(n). Self-balancing trees (AVL, red-black) maintain the guarantee on insert/delete.A részfák magassága legfeljebb konstanssal tér el, így a magasság O(log n) marad. Egy rendezett inputból épített kiegyensúlyozatlan BST láncolt listává fajul — a keresés O(n) lesz. Az önkiegyensúlyozó fák (AVL, piros-fekete) fenntartják a garanciát beszúráskor/törléskor.
DFS (recursive) and BFS (level-order)DFS (rekurzív) és BFS (szintenkénti)DFS O(n)/O(h) · BFS O(n)/O(w)
// DFS in-order — O(n) time, O(h) space
function inorder(node, out = []) {
  if (!node) return out;
  inorder(node.left, out);
  out.push(node.val);
  inorder(node.right, out);
  return out;
}

// BFS level-order — O(n) time, O(w) space (w = max width)
function levels(root) {
  if (!root) return [];
  const res = [], q = [root];
  let head = 0;
  while (head < q.length) {
    const size = q.length - head, level = [];
    for (let i = 0; i < size; i++) {
      const n = q[head++];
      level.push(n.val);
      if (n.left) q.push(n.left);
      if (n.right) q.push(n.right);
    }
    res.push(level);
  }
  return res;
}

Watch outBuktatóRecursive DFS on a deep tree can blow the call stack (~10k frames in Node) — mention an explicit-stack iterative version if the input could be large. And O(h) space is O(n) in the worst case, not O(log n), unless balance is guaranteed.A rekurzív DFS egy mély fán szétvetheti a hívási vermet (~10k keret Node-ban) — említs explicit-vermes iteratív verziót, ha az input nagy lehet. És az O(h) tár a legrosszabb esetben O(n), nem O(log n), hacsak nem garantált a kiegyensúlyozottság.

Heaps & Priority QueuesHeapek & prioritási sorok

A complete binary tree keeping the min (or max) at the root — O(log n) insert and extract, O(1) peek. The structure behind “top k” and “next smallest” problems.Egy teljes bináris fa, amely a minimumot (vagy maximumot) tartja a gyökérben — O(log n) beszúrás és kivétel, O(1) peek. A „top k” és „következő legkisebb” feladatok mögötti szerkezet.

peek O(1)peek O(1)insert/extract O(log n)beszúrás/kivétel O(log n)heapify O(n)heapify O(n)space O(n)tár O(n)

When is a heap the right answer?Mikor a heap a helyes válasz?

  • “Top k” / “k-th largest” (keep a size-k heap → O(n log k), better than sorting's O(n log n)), merging k sorted lists, running median (two heaps), and any scheduler picking the next-smallest item.„Top k” / „k-adik legnagyobb” (tarts k méretű heapet → O(n log k), jobb, mint a rendezés O(n log n)-je), k rendezett lista összefésülése, futó medián (két heap), és bármely ütemező, amely a következő legkisebb elemet választja.

Why does a size-k heap beat sorting?Miért veri a k méretű heap a rendezést?

  • Sorting does full work on all n elements: O(n log n). A heap capped at k only ever holds k items, so each of the n insertions costs O(log k): total O(n log k). When k ≪ n this is a large win, and space drops from O(n) to O(k).A rendezés teljes munkát végez mind az n elemen: O(n log n). Egy k-ra korlátozott heap sosem tart k-nál többet, így az n beszúrás mindegyike O(log k): összesen O(n log k). Ha k ≪ n, ez nagy nyereség, és a tár O(n)-ről O(k)-ra csökken.

Watch outBuktatóJavaScript has no built-in heap — say this out loud and either write a compact one or state that you'd use a library. Reaching for sort() on every insertion instead is O(n² log n) and a red flag.A JavaScriptben nincs beépített heap — mondd ki ezt hangosan, és vagy írj egy tömöret, vagy jelezd, hogy library-t használnál. Ha helyette minden beszúrásnál sort()-ot hívsz, az O(n² log n) és vörös zászló.

GraphsGráfok

Nodes connected by edges, directed or undirected, possibly weighted or cyclic. Many problems are graphs in disguise — grids, dependencies, state machines, social networks.Élekkel összekötött node-ok, irányított vagy irányítatlan, esetleg súlyozott vagy ciklikus. Sok feladat álcázott gráf — rácsok, függőségek, állapotgépek, közösségi hálók.

BFS/DFS O(V+E)BFS/DFS O(V+E)space O(V)tár O(V)adjacency list preferredszomszédsági lista ajánlott

Adjacency list vs matrix?Szomszédsági lista vs mátrix?

  • List (Map<node, node[]>): O(V+E) space, fast neighbor iteration — the default for sparse graphs, which most are. Matrix: O(V²) space but O(1) edge lookup — worth it only for dense graphs or constant-time edge queries.Lista (Map<node, node[]>): O(V+E) tár, gyors szomszéd-iteráció — az alapérték ritka gráfokhoz, amilyen a legtöbb. Mátrix: O(V²) tár, de O(1) él-lookup — csak sűrű gráfokhoz vagy konstans idejű él-lekérdezéshez éri meg.

BFS or DFS — how do you choose?BFS vagy DFS — hogyan választasz?

  • BFS explores by distance, so it finds the shortest path in an unweighted graph — use it whenever “fewest steps” appears. DFS goes deep first: cycle detection, topological sort, connected components, path existence, and backtracking.A BFS távolság szerint jár be, így megtalálja a legrövidebb utat súlyozatlan gráfban — használd, amikor a „legkevesebb lépés” felmerül. A DFS előbb mélyre megy: ciklus-észlelés, topologikus rendezés, összefüggő komponensek, út-létezés, és backtracking.

What about weighted shortest paths?Mi a helyzet a súlyozott legrövidebb utakkal?

  • BFS no longer works. Dijkstra (non-negative weights) with a priority queue is O((V+E) log V). Bellman-Ford handles negative weights in O(V·E) and detects negative cycles.A BFS már nem működik. A Dijkstra (nemnegatív súlyok) prioritási sorral O((V+E) log V). A Bellman-Ford kezeli a negatív súlyokat O(V·E)-ben és észleli a negatív ciklusokat.
BFS on a graph — shortest path in unweightedBFS gráfon — legrövidebb út súlyozatlanbanO(V+E) time / O(V) space
function bfs(graph, start, goal) {
  const visited = new Set([start]);
  const q = [[start, 0]];
  let head = 0;
  while (head < q.length) {
    const [node, dist] = q[head++];
    if (node === goal) return dist;
    for (const next of graph.get(node) ?? []) {
      if (!visited.has(next)) {
        visited.add(next);          // mark on ENQUEUE, not dequeue
        q.push([next, dist + 1]);
      }
    }
  }
  return -1;
}

Watch outBuktatóMark nodes visited when you enqueue, not when you dequeue — otherwise the same node enters the queue many times and the complexity degrades badly. Forgetting the visited set entirely means infinite loops on any cyclic graph.A node-okat enqueue-kor jelöld látogatottnak, ne dequeue-kor — különben ugyanaz a node sokszor bekerül a sorba, és a komplexitás súlyosan romlik. A visited halmaz teljes elfelejtése végtelen ciklust jelent bármely ciklikus gráfon.

Two PointersTwo pointers

Two indices moving through the data, usually from opposite ends or at different speeds — replacing a nested loop with a single pass. The first thing to try on a sorted array.Két index mozog az adaton át, általában ellentétes végekről vagy eltérő sebességgel — egy egymásba ágyazott ciklust egyetlen végigmenéssel helyettesítve. Rendezett tömbön ezt próbáld először.

O(n) timeO(n) időO(1) spaceO(1) tárneeds sorted input (often)rendezett input kell (gyakran)

When does it apply?Mikor alkalmazható?

  • A sorted array where you're looking for a pair/triplet matching a condition; palindrome checks; merging two sorted sequences; partitioning in place; and removing duplicates without extra space.Rendezett tömb, ahol feltételnek megfelelő párt/hármast keresel; palindróma-ellenőrzés; két rendezett sorozat összefésülése; helyben particionálás; és duplikátumok eltávolítása extra tár nélkül.

Why does moving from both ends work on sorted data?Miért működik a két végről indulás rendezett adaton?

  • Because the sum tells you which pointer to move: too small means you need a bigger value, so advance the left pointer; too large means shrink from the right. Each step eliminates a whole set of candidates, which is what buys the O(n).Mert az összeg megmondja, melyik mutatót mozgasd: ha túl kicsi, nagyobb érték kell, szóval léptesd a bal mutatót; ha túl nagy, szűkíts jobbról. Minden lépés egy egész jelölt-halmazt zár ki — ez adja az O(n)-t.
Two-sum on a sorted arrayTwo-sum rendezett tömbönO(n) time / O(1) space
function twoSumSorted(nums, target) {
  let l = 0, r = nums.length - 1;
  while (l < r) {
    const sum = nums[l] + nums[r];
    if (sum === target) return [l, r];
    if (sum < target) l++;      // need bigger
    else r--;                   // need smaller
  }
  return [];
}

Watch outBuktatóIf the input isn't sorted and you sort it yourself, the solution becomes O(n log n) and the original indices are lost — if the answer needs indices, a hash map in O(n) is the better trade.Ha az input nem rendezett és te rendezed, a megoldás O(n log n) lesz, és az eredeti indexek elvesznek — ha a válaszhoz indexek kellenek, egy hash map O(n)-ben jobb csere.

Sliding WindowSliding window

A window over a contiguous range that expands and contracts, maintaining a running state instead of recomputing — turning O(n·k) or O(n²) into O(n).Egy ablak egy összefüggő tartomány felett, amely tágul és szűkül, futó állapotot tartva újraszámítás helyett — O(n·k)-t vagy O(n²)-t alakítva O(n)-né.

O(n) timeO(n) időO(k) spaceO(k) tárcontiguous subarray/substringösszefüggő résztömb/részstring

Fixed vs variable window?Fix vs változó ablak?

  • Fixed: the size k is given — slide by adding the entering element and removing the leaving one. Variable: grow the right edge until the constraint breaks, then shrink from the left until it holds again, tracking the best window seen.Fix: a k méret adott — csúsztass a belépő elem hozzáadásával és a kilépő eltávolításával. Változó: növeld a jobb szélt, amíg a feltétel el nem törik, majd szűkíts balról, amíg újra teljesül, közben követve a legjobb látott ablakot.

What's the signal in a problem statement?Mi a jelzés a feladatszövegben?

  • The words contiguous, subarray or substring combined with longest/shortest/maximum under a constraint. If the elements needn't be adjacent, it's not a sliding window.Az összefüggő, résztömb vagy részstring szavak kombinálva a leghosszabb/legrövidebb/maximum feltétellel. Ha az elemeknek nem kell szomszédosnak lenniük, az nem sliding window.
Longest substring without repeating charactersLeghosszabb részstring ismétlődő karakter nélkülO(n) time / O(min(n,alphabet)) space
function longestUnique(s) {
  const lastSeen = new Map();
  let best = 0, start = 0;
  for (let end = 0; end < s.length; end++) {
    const c = s[end];
    // only jump forward — never move start backwards
    if (lastSeen.has(c) && lastSeen.get(c) >= start) {
      start = lastSeen.get(c) + 1;
    }
    lastSeen.set(c, end);
    best = Math.max(best, end - start + 1);
  }
  return best;
}

Watch outBuktatóThe left pointer must only ever move forward — resetting or moving it backwards breaks the O(n) guarantee and usually the correctness too. Note the >= start guard above: a stale index from before the window must be ignored.A bal mutató csak előre mozoghat — visszaállítása vagy visszaléptetése megtöri az O(n) garanciát, és általában a helyességet is. Figyeld a fenti >= start őrt: egy elavult, ablak előtti indexet figyelmen kívül kell hagyni.

Binary SearchBináris keresés

Halving the search space each step — O(log n). Applies to sorted arrays, but more powerfully to any monotonic predicate, including searching over an answer range.A keresési tér felezése minden lépésben — O(log n). Rendezett tömbökre alkalmazható, de erősebben bármely monoton predikátumra, beleértve a válasz-tartomány feletti keresést.

O(log n) timeO(log n) időO(1) spaceO(1) tárneeds monotonicitymonotonitás kell

What is “binary search on the answer”?Mi a „bináris keresés a válaszon”?

  • When the answer is a number in a range and you can cheaply test “is x feasible?”, and feasibility is monotonic (once true, always true above), binary search the range instead of the array. Classic uses: minimum capacity, minimum days, minimum speed.Amikor a válasz egy szám egy tartományban, és olcsón tesztelhető, hogy „megvalósítható-e x?”, és a megvalósíthatóság monoton (ha egyszer igaz, felette mindig igaz), a tömb helyett a tartományon keress binárisan. Klasszikus használat: minimális kapacitás, minimális napok, minimális sebesség.

How do you avoid the classic off-by-one and infinite loops?Hogyan kerülöd el a klasszikus off-by-one-t és a végtelen ciklust?

  • Pick one template and keep it. With while (l < r) and mid = (l+r)>>1, always move at least one pointer past mid (l = mid+1), or the loop can stall when l and r are adjacent.Válassz egy sablont és tartsd magad hozzá. A while (l < r) és mid = (l+r)>>1 mellett mindig mozgass legalább egy mutatót mid-en túlra (l = mid+1), különben a ciklus megakadhat, amikor az l és r szomszédos.
Leftmost insertion point (lower bound)Bal szélső beszúrási pont (lower bound)O(log n) time / O(1) space
function lowerBound(nums, target) {
  let l = 0, r = nums.length;          // r is exclusive
  while (l < r) {
    const mid = (l + r) >> 1;
    if (nums[mid] < target) l = mid + 1;
    else r = mid;                      // keep mid as a candidate
  }
  return l;   // first index with nums[i] >= target
}

Watch outBuktatóIn other languages (l+r)/2 can overflow; in JS the real trap is different — (l+r)>>1 uses 32-bit ints, so it breaks above ~2³¹. Use Math.floor((l+r)/2) or l + ((r-l)>>1) for very large ranges (common in binary-search-on-answer problems).Más nyelvekben az (l+r)/2 túlcsordulhat; JS-ben a valódi csapda más — az (l+r)>>1 32 bites egészeket használ, szóval ~2³¹ felett elromlik. Nagyon nagy tartományokhoz (gyakori a válaszon-bináris-keresés feladatoknál) használj Math.floor((l+r)/2)-t vagy l + ((r-l)>>1)-et.

Recursion & BacktrackingRekurzió & backtracking

Solving a problem by solving smaller instances of itself; backtracking adds “try a choice, recurse, undo it” to explore a search space systematically.Egy feladat megoldása önmaga kisebb példányainak megoldásával; a backtracking hozzáadja a „próbálj egy választást, rekurzálj, vond vissza” mintát, hogy szisztematikusan bejárj egy keresési teret.

often O(2ⁿ) or O(n!)gyakran O(2ⁿ) vagy O(n!)O(depth) call stackO(mélység) hívási veremprune to survivevágj, hogy túléld

What are the parts of a correct recursion?Mik egy helyes rekurzió részei?

  • A base case that returns without recursing, a recursive case that moves strictly toward the base case, and correct state handling — either pass copies or mutate and undo.Egy báziseset, amely rekurzió nélkül tér vissza, egy rekurzív eset, amely szigorúan a báziseset felé halad, és helyes állapotkezelés — vagy másolatokat adsz át, vagy mutálsz és visszavonsz.

What problems are backtracking?Mely feladatok backtrackingek?

  • Generating permutations, combinations and subsets; N-queens; sudoku; word search on a grid; partitioning. The giveaway is “find all …” or “generate every valid …”.Permutációk, kombinációk és részhalmazok generálása; N-királynő; sudoku; szókeresés rácson; particionálás. Az árulkodó jel a „találd meg az összes…” vagy „generáld minden érvényes…”.
Subsets via backtrackingRészhalmazok backtrackinggelO(n·2ⁿ) time / O(n) space (excl. output)
function subsets(nums) {
  const res = [], path = [];
  function backtrack(start) {
    res.push([...path]);              // copy — path keeps mutating
    for (let i = start; i < nums.length; i++) {
      path.push(nums[i]);             // choose
      backtrack(i + 1);               // explore
      path.pop();                     // un-choose
    }
  }
  backtrack(0);
  return res;
}

Watch outBuktatóPushing path instead of [...path] stores a reference to an array that keeps mutating — every entry in the result ends up identical (usually empty). And forgetting the pop() leaks state into sibling branches.Ha a path-t pusholod a [...path] helyett, egy folyamatosan mutálódó tömbre való referenciát tárolsz — az eredmény minden eleme azonos lesz (általában üres). És a pop() elfelejtése átszivárogtatja az állapotot a testvér-ágakba.

Dynamic ProgrammingDinamikus programozás

Solving overlapping subproblems once and reusing the results — either top-down with memoization or bottom-up with a table. It turns exponential recursion into polynomial time.Átfedő részfeladatok egyszeri megoldása és az eredmények újrafelhasználása — vagy top-down memoizációval, vagy bottom-up táblával. Exponenciális rekurziót alakít polinomiális idejűvé.

typically O(states × transition)tipikusan O(állapotok × átmenet)O(states), often reducibleO(állapotok), gyakran csökkenthető

When is DP applicable?Mikor alkalmazható a DP?

  • Two conditions: optimal substructure (the optimal answer is built from optimal answers to subproblems) and overlapping subproblems (the same subproblem recurs). Without overlap, plain recursion or greedy is enough.Két feltétel: optimális részszerkezet (az optimális válasz részfeladatok optimális válaszaiból épül) és átfedő részfeladatok (ugyanaz a részfeladat ismétlődik). Átfedés nélkül elég a sima rekurzió vagy a mohó megoldás.

Memoization vs tabulation?Memoizáció vs tabuláció?

  • Memoization (top-down): write the natural recursion, cache results — easier to derive, but uses the call stack. Tabulation (bottom-up): fill a table iteratively — no stack risk and often allows space optimization.Memoizáció (top-down): írd meg a természetes rekurziót, cache-eld az eredményeket — könnyebb levezetni, de a hívási vermet használja. Tabuláció (bottom-up): töltsd a táblát iteratívan — nincs verem-kockázat és gyakran engedi a tár-optimalizálást.

How do you approach a DP problem in an interview?Hogyan állsz neki egy DP feladatnak interjún?

  • Define the state in words first (“dp[i] = the best answer using the first i items”), write the recurrence, identify the base cases, then decide the iteration order. Start with brute-force recursion and add memoization — deriving the table directly is harder and error-prone.Előbb definiáld az állapotot szavakban („dp[i] = a legjobb válasz az első i elemmel”), írd fel a rekurziót, azonosítsd a báziseseteket, majd döntsd el az iterációs sorrendet. Indulj nyers erejű rekurzióval és add hozzá a memoizációt — a tábla közvetlen levezetése nehezebb és hibalehetőség.
Climbing stairs — three stages of the same ideaLépcsőzés — ugyanaz az ötlet három szakaszbanO(2ⁿ) → O(n)/O(n) → O(n)/O(1)
// 1. naive recursion — O(2ⁿ)
const naive = n => n <= 2 ? n : naive(n-1) + naive(n-2);

// 2. memoized — O(n) time, O(n) space
function memo(n, cache = new Map()) {
  if (n <= 2) return n;
  if (cache.has(n)) return cache.get(n);
  const r = memo(n-1, cache) + memo(n-2, cache);
  cache.set(n, r);
  return r;
}

// 3. tabulated + space-optimized — O(n) time, O(1) space
function best(n) {
  if (n <= 2) return n;
  let a = 1, b = 2;
  for (let i = 3; i <= n; i++) [a, b] = [b, a + b];
  return b;
}

Watch outBuktatóOnly the last row (or two) of a DP table is usually needed — say “I can reduce space from O(n·m) to O(m)” even if you don't implement it; interviewers listen for that. And a memo key must capture every varying parameter, or you'll return cached answers for different states.Egy DP-táblából általában csak az utolsó sor (vagy kettő) kell — mondd ki, hogy „a tárat O(n·m)-ről O(m)-re tudom csökkenteni”, akkor is, ha nem implementálod; az interjúztatók erre figyelnek. És a memo-kulcsnak minden változó paramétert meg kell fognia, különben eltérő állapotokra adsz vissza cache-elt válaszokat.

Greedy AlgorithmsMohó algoritmusok

Making the locally optimal choice at each step and never reconsidering — fast and simple when it works, but it needs proof that local optimality yields a global optimum.A lokálisan optimális választás minden lépésben, újragondolás nélkül — gyors és egyszerű, amikor működik, de bizonyítani kell, hogy a lokális optimalitás globális optimumot ad.

often O(n log n) (sort + pass)gyakran O(n log n) (rendezés + végigmenés)wrong without the exchange argumenthibás az exchange argument nélkül

Greedy vs DP — how do you tell?Greedy vs DP — hogyan döntöd el?

  • If a locally best choice can be proven never to hurt the final answer (an exchange argument), greedy works and is cheaper. If a choice's value depends on future choices, you need DP. Coin change with arbitrary denominations is the classic case where greedy fails and DP is required.Ha egy lokálisan legjobb választásról bizonyítható, hogy sosem ront a végső válaszon (exchange argument), a greedy működik és olcsóbb. Ha egy választás értéke a jövőbeli választásoktól függ, DP kell. A tetszőleges címletű pénzváltás a klasszikus eset, ahol a greedy elbukik és DP kell.

Typical greedy problems?Tipikus greedy feladatok?

  • Interval scheduling (sort by end time), merging intervals (sort by start), jump game, assigning tasks by deadline, and Huffman coding. Nearly all begin with a sort that establishes the greedy order.Intervallum-ütemezés (végidő szerint rendezve), intervallumok összevonása (kezdet szerint rendezve), jump game, taszkok kiosztása határidő szerint, és Huffman-kódolás. Szinte mind egy rendezéssel kezdődik, amely megállapítja a mohó sorrendet.
Merge overlapping intervalsÁtfedő intervallumok összevonásaO(n log n) time / O(n) space
function merge(intervals) {
  if (!intervals.length) return [];
  intervals.sort((a, b) => a[0] - b[0]);        // by start
  const out = [intervals[0].slice()];
  for (const [s, e] of intervals.slice(1)) {
    const last = out[out.length - 1];
    if (s <= last[1]) last[1] = Math.max(last[1], e);  // overlap
    else out.push([s, e]);
  }
  return out;
}

Watch outBuktatóNever assert a greedy solution without justifying why the local choice is safe — “it seems to work on the examples” is exactly how greedy fails. If you can't argue it, reach for DP.Sose állíts mohó megoldást annak indoklása nélkül, hogy miért biztonságos a lokális választás — a „a példákon működni látszik” pontosan az, ahogy a greedy elbukik. Ha nem tudod megindokolni, nyúlj a DP-hez.

The sequenceA sorrend

  • 1. Restate the problem and confirm with an example. 2. Ask about constraints — input size, value ranges, duplicates, empty input, sorted? 3. State a brute-force approach with its complexity. 4. Which constraints should you always ask about?Mely megkötésekről kérdezz mindig?

  • Input size (decides whether O(n²) is acceptable), value ranges (enables counting sort or bit tricks), duplicates, negative numbers, whether the input is sorted, and whether you may mutate it. What edge cases apply to almost everything?Mely peremesetek vonatkoznak szinte mindenre?

  • Empty input, a single element, all elements identical, already-sorted and reverse-sorted input, negative numbers and zero, and integer overflow on large sums. Mentioning these unprompted is a strong signal.Üres input, egyetlen elem, minden elem azonos, már rendezett és fordítva rendezett input, negatív számok és nulla, és egész-túlcsordulás nagy összegeknél. What if you're stuck?Mi van, ha elakadsz?

  • Say what you're considering out loud — the interviewer can only help if they know where you are. Próbálj egy kisebb példát kézzel, kérdezd meg, elfogadható-e egy egyszerűbb változat, vagy nevezd meg a mintát, amire gyanakszol. A csend az egyetlen visszafordíthatatlan lépés.
